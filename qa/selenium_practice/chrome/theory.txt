ВИДЕО-1

Для работы с selenium нужно установить библиотеку selenium.
Для различных браузеров нужно установить различные драйвера.
Для работы с драйвером нужно создать экземпляр класса.
Finally при try-except обязателен, иначе появятся лишние процессы в ОС.
Открыть браузер можно через driver.get.


ВИДЕО-2

User-agent - сайты и сервисы собирают много информации о пользователях. User-agent содержит информацию об устройстве, операционной системе, с которой
пользователь посещает сайт. С учетом полученного, сайты принимают решение о корректном отображении информации на нем.
Подмена User-agent полезна с целью сохранения анонимности или посылания автоматических запросов.
Для подмены нужно создать объект опций. User agent можно поменять даже для мобильных устройств. Также, можно создать списки
из различных user-agent. Существует либа для подмены user-agent: fake user-agent.


ВИДЕО-3

В selenium также можно подключить прокси. Подключить прокси можно в одну строку. Передать ip и порт можно через options.
Важно то, что для авторизации возможно поставить selenium-wire.


ВИДЕО-4

Авторизацию можно реализовать через метод find_element. Отправка данных через send_keys. Кнопки также имеют свой id. В selenium
есть функция click для выполнения функционала кнопок.


ВИДЕО-5

Сохранять и подгружать куки, используя драйвер selenium, также возможно. Делать это можно, например, для того чтобы не проходить
лишний раз аутентификацию. Сохранить куки можно используя модуль pickle, добавив туда функцию get_cookies. add_cookies - для
загрузки куки. Можно загрузить из файла.


ВИДЕО-6

Режим WEB-driver можно также отключить. Отключение этого режима позволяет автоматическому тестирующему запросу быть похожим
на пользователя. Для старых версий (до 79, это где-то 19 год) в хроме все делалось по другому...
Для новых версий достаточно аргумента --disable0blink-features=AutomationControlled.


ВИДЕО-7

В фоновом режиме также можно запускать браузер. Для этого нужен headless mod. Либо через опции, либо через метод класса.
